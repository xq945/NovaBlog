# 设计文档：JWT 认证

## 1. 需求概述

- **功能名称**：JWT 认证
- **目标**：实现基于 JWT 的接口访问控制，登录成功后返回 Token，后续请求自动携带 Token 进行身份验证
- **涉及角色**：所有需要登录的接口（文章发布、个人中心等）

## 2. 接口设计

### 2.1 接口变更

| 接口 | 变更类型 | 说明 |
|------|----------|------|
| POST /user/login | 修改 | 响应增加 token 字段 |
| POST /auth/refresh | 新增 | 使用 Refresh Token 换取新的 Access Token |

### 2.2 登录响应（改造后）

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "token": "eyJhbGciOiJIUzI1NiIs...",
    "refreshToken": "eyJhbGciOiJIUzI1NiIs...",
    "expiresIn": 7200,
    "userInfo": {
      "id": 1,
      "username": "zhangsan",
      "nickname": "张三",
      "avatar": null,
      "email": null,
      "role": "USER"
    }
  }
}
```

- **token**：Access Token，有效期 2 小时，用于接口访问
- **refreshToken**：Refresh Token，有效期 7 天，用于刷新 Access Token
- **expiresIn**：Access Token 过期时间（秒）
- **userInfo**：用户信息（不含密码）

### 2.3 Refresh Token 刷新接口

```http
POST /auth/refresh
```

- **说明**：该接口**不走 JWT 拦截器**（放行），由 AuthController 自己从请求头提取 Refresh Token 并验证
- **请求头**：`Authorization: Bearer {refreshToken}`
- **响应（成功）**：

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "token": "eyJhbGciOiJIUzI1NiIs...",
    "refreshToken": "eyJhbGciOiJIUzI1NiIs...",
    "expiresIn": 7200
  }
}
```

- **响应（失败）**：Refresh Token 过期或无效，返回 401，要求重新登录

### 2.4 受保护接口规范

需要登录的接口，请求头必须携带：
```
Authorization: Bearer {token}
```

**放行路径（无需 Token）**：
| 路径 | 说明 |
|------|------|
| POST /user/register | 用户注册 |
| POST /user/login | 用户登录 |
| POST /auth/refresh | Token 刷新 |
| GET /article/list | 文章列表 |
| GET /article/{id} | 文章详情 |
| GET /comment/list | 评论列表 |

**其他路径默认需要 Token**。

未携带 Token 或 Token 过期返回（HTTP 状态码 200，业务 code 401）：
```json
{
  "code": 401,
  "message": "Token 无效或已过期",
  "data": null
}
```

## 4. 核心逻辑

### 4.1 JwtProperties 配置类

使用 `@ConfigurationProperties(prefix = "jwt")` 读取配置：

| 字段 | 类型 | 说明 |
|------|------|------|
| secret | String | JWT 签名密钥 |
| expiration | Long | Access Token 过期时间（毫秒）|
| refreshExpiration | Long | Refresh Token 过期时间（毫秒）|

### 4.2 Token 生成（JwtUtil）

```
生成 Access Token
  ↓
载荷：{ userId, username, role, iat(签发时间), exp(过期时间) }
  ↓
使用 HMAC-SHA256 签名（密钥来自 JwtProperties）
  ↓
返回 JWT 字符串
```

```
生成 Refresh Token
  ↓
载荷：{ userId, type: "refresh", iat(签发时间), exp(7天后) }
  ↓
使用同一密钥签名
  ↓
返回 JWT 字符串
```

### 4.3 Token 校验流程（JWT 拦截器）

实现 `HandlerInterceptor` 接口：

```
preHandle（请求到达）
  ↓
从请求头提取 Authorization: Bearer {token}
  ↓
Token 是否存在？
  ├─ 否 → 返回 401
  ↓
解析 Token
  ↓
签名是否有效？
  ├─ 否 → 返回 401
  ↓
是否过期？
  ├─ 是 → 返回 401（提示使用 Refresh Token）
  ↓
将用户信息（userId, username, role）存入 ThreadLocal
  ↓
放行请求

afterCompletion（请求结束）
  ↓
清理 ThreadLocal（必须！防止内存泄漏）
```

### 4.4 登录接口改造流程

```
校验用户名密码
  ↓
生成 Access Token（2h）
  ↓
生成 Refresh Token（7d）
  ↓
返回 { token, refreshToken, expiresIn, userInfo }
```

### 4.5 UserContext 工具类

封装 ThreadLocal 操作，供拦截器和 Controller 使用：

```java
public class UserContext {
    private static final ThreadLocal<UserDTO> HOLDER = new ThreadLocal<>();
    public static void set(UserDTO user) { HOLDER.set(user); }
    public static UserDTO get() { return HOLDER.get(); }
    public static Long getUserId() { return get() != null ? get().getId() : null; }
    public static void clear() { HOLDER.remove(); }
}
```

### 4.6 异常处理

| 异常场景 | 处理方式 | 返回结果 |
|----------|----------|----------|
| 未携带 Token | 拦截器拦截 | Result.error(401, "请先登录") |
| Token 格式错误 | 拦截器拦截 | Result.error(401, "Token 格式错误") |
| Token 签名无效 | 拦截器拦截 | Result.error(401, "Token 无效") |
| Token 过期 | 拦截器拦截 | Result.error(401, "Token 已过期，请重新登录") |
| Refresh Token 过期/无效 | AuthController 校验 | Result.error(401, "登录已过期，请重新登录") |

## 5. 安全考虑

- [x] 密钥存储：从 `application.yml` / 环境变量读取，不硬编码
- [x] 密钥长度：>= 256 位（32 字节），当前配置 79 字符
- [x] Token 过期：Access Token 2 小时，Refresh Token 7 天
- [x] 敏感信息：Token 载荷不存放密码
- [x] 传输安全：HTTPS 生产环境（开发环境可 HTTP）
- [x] ThreadLocal 清理：请求结束后必须清理，防止内存泄漏
- [x] 生产环境密钥：通过环境变量 `NOVABLOG_JWT_SECRET` 注入，不依赖 `application-dev.yml`

## 6. 前端交互

### 6.1 登录成功后

```
收到响应
  ↓
保存 token 到 localStorage
  ↓
保存 refreshToken 到 localStorage
  ↓
保存 userInfo 到 Pinia Store
  ↓
关闭弹窗，更新导航栏显示用户信息
```

### 6.2 页面刷新后恢复登录状态

App.vue 中应用启动时检查 localStorage：

```javascript
onMounted(() => {
  const token = localStorage.getItem('token')
  if (token) {
    userStore.token = token
    userStore.refreshToken = localStorage.getItem('refreshToken') || ''
    // 可选：调接口 /user/profile 获取用户信息
  }
})
```

### 6.3 退出登录

导航栏点击退出按钮后：

```
点击退出
  ↓
调用 userStore.clearToken()
  ↓
清除 localStorage 中的 token / refreshToken
  ↓
清除 Pinia 中的 userInfo
  ↓
刷新页面恢复未登录状态
```

### 6.4 首页监听全局登录事件

HomeView.vue 中监听 `show-login` 事件，显示登录弹窗：

const handleShowLogin = (event) => {

onMounted(() => {

### 6.5 Axios 请求拦截器改造

```javascript
// 请求拦截器：自动携带 Token
request.interceptors.request.use(config => {
  const token = localStorage.getItem('token')
  if (token) {
    config.headers.Authorization = 'Bearer ' + token
  }
  return config
})

// 响应拦截器：Token 过期处理
request.interceptors.response.use(
  response => response.data,
  error => {
    const res = error.response?.data
    if (res?.code === 401) {
      // 清除登录状态
      userStore.clearToken()
      // 触发自定义事件，通知首页显示登录弹窗
      window.dispatchEvent(new CustomEvent('show-login', {
        detail: { message: res.message || '登录已过期，请重新登录' }
      }))
    }
    return Promise.reject(error)
  }
)
```

### 6.6 Pinia Store 改造

```javascript
const useUserStore = defineStore('user', () => {
  const token = ref(localStorage.getItem('token') || '')
  const refreshToken = ref(localStorage.getItem('refreshToken') || '')
  const userInfo = ref(null)

  const setToken = (newToken, newRefreshToken) => {
    token.value = newToken
    refreshToken.value = newRefreshToken
    localStorage.setItem('token', newToken)
    localStorage.setItem('refreshToken', newRefreshToken)
  }

  const clearToken = () => {
    token.value = ''
    refreshToken.value = ''
    userInfo.value = null
    localStorage.removeItem('token')
    localStorage.removeItem('refreshToken')
  }

  return { token, refreshToken, userInfo, setToken, clearToken }
})
```
